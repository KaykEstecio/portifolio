# Frontend Visual UI

## Objetivo

Melhorar interfaces frontend para que pareçam mais profissionais, modernas, responsivas e com identidade própria.

A prioridade é evitar aparência genérica de tutorial, sem deixar o código complexo demais.

## Regras

- Não criar interface com aparência genérica.
- Não usar apenas fundo branco, título centralizado e botão padrão.
- Melhorar hierarquia visual, espaçamento, alinhamento e contraste.
- Criar identidade visual consistente.
- Manter responsividade para celular, tablet e desktop.
- Não quebrar funcionalidades existentes.
- Não alterar regra de negócio sem necessidade.
- Não criar animações exageradas.
- Manter o código simples e fácil de explicar.

## Para projeto de hamburgueria

Quando o projeto for de hamburgueria, restaurante ou delivery:

- Usar visual mais quente e chamativo.
- Priorizar cores como vermelho, amarelo, laranja, preto, marrom, bege ou tons escuros.
- Criar cards de produto com imagem, nome, descrição curta, preço e botão de ação.
- Melhorar navbar, footer, página inicial, login, cadastro, cardápio e carrinho.
- Fazer a interface parecer uma marca real, não um template acadêmico genérico.
- Evitar aparência de dashboard administrativo na área do cliente.

## Checklist antes de finalizar

Antes de considerar a alteração pronta, verificar:

1. A tela ficou menos genérica?
2. A interface tem identidade visual clara?
3. O usuário entende rapidamente o que fazer?
4. Os botões principais se destacam?
5. Os cards estão bem organizados?
6. A navbar parece profissional?
7. O footer está organizado?
8. A tela funciona no celular?
9. A tela funciona no desktop?
10. O código continua simples?

## Resposta esperada

Ao finalizar, explicar:

- Quais arquivos foram alterados.
- O que melhorou visualmente.
- Por que a interface ficou menos genérica.
- Como testar no navegador.
