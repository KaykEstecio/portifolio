03 / Projetos

Projetos em destaque

Projetos acadêmicos e de estudo apresentados com foco em backend, regras de negócio, autenticação, dados, documentação e deploy.

Sistema Web para Hamburgueria

NANDS Burger

Problema
Uma hamburgueria precisa organizar produtos, pedidos e acesso administrativo em um sistema que possa ser apresentado como projeto acadêmico e evoluir para uso real.
Solução
Construí um sistema web com cadastro de produtos, fluxo de pedidos, autenticação e área administrativa, conectando interface, rotas da aplicação e persistência de dados.
O que implementei
  • Estrutura de rotas para operações do sistema
  • Modelagem inicial de dados com Prisma
  • Fluxo de autenticação para separar acesso comum e administrativo
  • Organização de produtos e pedidos para o painel administrativo
  • Publicação do projeto na Vercel e documentação no GitHub
Resultado
Demonstra que consigo transformar uma regra de negócio simples em uma aplicação web estruturada, com preocupação em dados, autenticação, deploy e documentação.

Tecnologias

  • Node.js
  • Prisma
  • JavaScript
  • Banco de dados
  • Vercel

02

Aplicação web backend

Sistema de Aluguel de Veículos

Problema
Um sistema de reservas precisa controlar usuários, veículos disponíveis e operações de aluguel sem perder a organização das regras de negócio.
Solução
Desenvolvi uma aplicação web para simular cadastro, autenticação, catálogo de veículos e fluxo de reservas, aproximando conceitos de backend e banco de dados.
O que implementei
  • Rotas para cadastro, login e navegação do sistema
  • Operações de CRUD para organizar dados da aplicação
  • Validação básica das informações enviadas pelo usuário
  • Templates HTML/CSS integrados ao backend Flask
  • Separação inicial das regras de reserva e controle de veículos
Resultado
Mostra minha base em desenvolvimento web com Python, organização de rotas, autenticação, CRUD e aplicação de regras de negócio em um contexto prático.

Tecnologias

  • Python
  • Flask
  • SQL
  • HTML
  • CSS

03

Projeto de lógica e dados

Controle de Gastos

Problema
Registrar despesas manualmente dificulta acompanhar categorias, entradas e saídas de forma organizada.
Solução
Criei um projeto em Python para registrar gastos, validar entradas, organizar registros e praticar manipulação de dados com uma estrutura simples.
O que implementei
  • Entrada e validação de dados no fluxo do programa
  • Organização de registros por categoria
  • Funções para separar responsabilidades do código
  • Tratamento básico de informações financeiras
  • Estrutura pensada para futura integração com banco de dados
Resultado
Demonstra fundamentos importantes para backend: lógica, validação, organização de dados e evolução gradual de um script para uma aplicação mais completa.

Tecnologias

  • Python
  • Lógica de programação
  • Manipulação de dados
